Features

Everything you need to caption the world around you

Purpose-built for watching foreign-language videos, following meetings, and making the screen more accessible — without handing your audio to a cloud service.

Mic & system audio

Caption whatever the mic hears, or capture audio from video apps using Android's MediaProjection API — no root required.

Streaming on-device recognition

One long-lived Vosk recognizer is fed ~100 ms audio chunks continuously, so words appear as you speak them — no 2-second batch delay, no cloud round-trip. Works the same for mic and system audio.

Any language your server supports

LiveCaptionN reads /languages from your LibreTranslate endpoint and populates the pickers dynamically — add more on the server, they show up in the app.

Floating overlay

Draggable, resizable, always-on-top captions with adjustable text size, opacity, and pause/minimize controls.

Transcript history

Every caption is saved to a local transcript log you can revisit, copy, or clear — persisted in a JSON file on-device.

Private by default

No analytics, no ads, no cloud lock-in. Run it against your own self-hosted servers or stay fully offline with Vosk.

Automatic update notifications

A background WorkManager job polls the GitHub Releases API twice a day. When a new build ships, you get a system notification with a one-tap Download action — and an in-app banner the next time you open the app.

Languages & translation

Two ways to translate — pick whichever fits your setup

LiveCaptionN is a two-stage pipeline: speech → text runs in a speech engine, then text → text runs in LibreTranslate. The language pickers adapt to whichever path you choose so you only ever see options that actually work.

Path A — LibreTranslate

Point the app at any LibreTranslate-compatible server. On launch it calls GET /languages and populates both Source and Target pickers with whatever the server reports — typically 25–35+ languages depending on which Argos Translate packages are installed.

Want more languages? SSH into your LibreTranslate host and install extra packages:

argospm update
argospm install translate-en_ja translate-en_ko translate-en_fa
# …then restart LibreTranslate

Change the base URL in-app and LiveCaptionN refetches the list automatically.

Path B — On-device Vosk

Select System Audio → Local Vosk and the source-language picker collapses to just the Vosk models installed on this phone. Two models ship inside the APK (English and Vietnamese) so it works out of the box with zero downloads.

Tap Manage on-device models to grab more (≈30–80 MB each): Spanish, French, German, Italian, Portuguese, Dutch, Russian, Ukrainian, Polish, Czech, Turkish, Arabic, Persian, Hindi, Chinese, Japanese, Korean, Indonesian. Models are fetched from alphacephei.com/vosk/models over HTTPS and unzipped to app-private storage.

On-device transcription still uses LibreTranslate for the text → text step. If the source and target match (English speech → English captions), no translation call is made.

How it works

Sideload the APK from the latest GitHub release, then allow microphone access and the Display over other apps permission.

Pick audio source and backend

FAQ

Answers for the curious

Is LiveCaptionN free?

Yes. It's fully open source under an MIT-compatible license, has no ads, collects no analytics, and every release APK is built by GitHub Actions from a public commit you can inspect.

Which Android versions are supported?

Android 10 (API 29) and above. Some features like createConfigForDefaultDisplay() for MediaProjection are used on Android 14+ where available.

Do I need my own Whisper server?

For the remote Whisper backend, yes — any Whisper-compatible HTTP endpoint works (e.g. faster-whisper-server, whisper-asr-webservice). If you don't want to run a server, switch to the on-device Vosk backend instead.

What about translation?

Translation goes through any LibreTranslate-compatible endpoint. Self-host LibreTranslate or point at a public instance you trust. The app fetches GET /languages from whatever server you configure and populates the language pickers from that list — so whichever Argos Translate packages are installed on your server is exactly what you'll see in the app.

How do I add more languages?

For translation, install extra Argos Translate packages on your LibreTranslate server (argospm install translate-en_ja, etc.) and the app will pick them up on the next refresh. For on-device speech recognition, tap Manage on-device models inside the app to download additional Vosk models for Spanish, French, German, Russian, Chinese, Japanese, Korean, and many more.

Does on-device mode need internet?

Only for the translation step. Speech recognition via Vosk is fully offline once the model is downloaded. If your source and target language match (e.g. English speech → English captions), no translation call is made and the app works entirely offline.

Can it caption YouTube or TikTok directly?

Yes, via the system-audio mode that uses Android's MediaProjection + AudioPlaybackCapture. Note: Android only allows capturing audio from apps that permit playback capture, so some DRM-protected streams will not be captured.

Where are my captions stored?

Locally, in a JSON file inside the app's private storage. Nothing leaves the device except the speech sent to your ASR and translation endpoints.
